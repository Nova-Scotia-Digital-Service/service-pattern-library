Content page

Your main content area shouldn't exceed 800px, lines of text that span the screen are hard to read. If you don't have a sidebar, use col-8to restrict the column to a readable length. Don't set the column width to full-width col-12.

Page title

Each page in your service must contain a distinct page title tag. Page titles should include the page heading, the name of the service, and Government of Nova Scotia. As an example, the page title of this page is <title>Content Page - Forms & Services Building Blocks - Government of Nova Scotia</title>

Consideration for bilingual services

In a bilingual service, all the French page should have the page heading in French, the name of the service in French, and « Gouvernement de la Nouvelle-Écosse ».

<title>Modèle de page de contenu - Bibliothèque de configurations de conception de service - Gouvernement de la Nouvelle-Écosse</title>

Skip link

Include a skip link component to give users the option to bypass the top-level navigation links and jump to the main content on a page.

Header and main navigation

Use the "NS MOTHERSHIP RIBBON" provided in the CNS saltire pattern library.

Only include a search bar or a help link if user testing shows they make the service easier to use. Instead, try adding information to the pages where needed. Use the accordion compontent (link not available yet) to hold these pieces of information.

Consideration for bilingual services

The English pages should have a « Français » language toggle and use the English Nova Scotia logo. The French pages should have a "English" language toggle and use the bilingual Nova Scotia logo.

Page heading

There should only be one H1 heading on each page. Don't use the same page heading across multiple pages. For example, don't use the H1 heading to show the name of the service across multiple pages.

The page heading should relate to the content of the current page, not any higher-level section. If you need to show the high-level section, use the ns-caption style. An example of this is multiple forms within one service. When testing shows confusion about which form the user is on, use the ns-caption style to show the name of the form in the caption.

Sub-heading

Sub-headings must not skip heading levels. If a sub-heading visually looks too big, change the size by adding a heading class for the desired level.

Footer

Use the footer provided in the CNS saltire pattern library. Remove the social media links unless there's a benefit to display them. Don't change the links to privacy, terms and cookies. These link to provincial standards should be the same for all services. If your service has it's own provacy notice, place it on the page where private information is being collected. Use the privacy notice pattern to hold the information.

Consideration for bilingual services

In a bilingual service, all French pages should have a French footer. Use the footer-fr.html file from the CNS saltire pattern library.
