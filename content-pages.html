Content page

Your main content area shouldn't exceed 800px, lines of text that span the screen are hard to read. If you don't have a sidebar, use col-8to restrict the column to a readable length. Don't set the column width to full-width col-12.

Basic Layout

Page title

Each page in your service must contain a distinct page title tag. Page titles should include the page heading, the name of the service, and Government of Nova Scotia. As an example, the page title of this page is <title>Content Page - Service Pattern Library - Government of Nova Scotia</title>

Header and main navigation

Use the "NS MOTHERSHIP RIBBON" provided in the CNS saltire pattern library. In bilingual services, when the service toggles to the French, use the French version on the ribbon. Don't forget to change the page language declaration <html lang="fr"> and update any aria labels and alt text on the page.

Only include a search bar or a help link if user testing shows they make the service easier to use. Instead, try adding information to the pages where needed. Use the accordion compontent (link not available yet) to hold these pieces of information.

Page heading

Don't use the same page heading across multiple pages.

The page heading should relate to the content of the current page, not any higher-level section. If you need to show the high-level section, use the ns-caption style.

Sub-heading

Sub-headings must not skip heading levels. If a sub-heading visually looks too big, change the size by adding a heading class for the desired level.

Footer

Use the footer provided in the CNS saltire pattern library. Remove the social media links unless there's a benefit to display them.
